overview

What each side is doing

The page is optimized for engineers skimming architecture, not stakeholders browsing a pitch deck.

claude

planning and orchestration

Best used for decomposition, file mapping, sequencing, and coordinating multiple agents or tasks.

codex

review and execution support

Useful for defensive critique, bounded execution, and parallel work when tasks are isolated enough.

human

approval and correction

The process is stronger when the user approves the plan and resolves conflicts before code starts moving.

Planning and review are separated.

Claude can optimize for structure while Codex optimizes for scrutiny. That separation is the point.

Hooks are stronger than loose prompt rules.

They fire deterministically and are better for gates, audits, and safety checks.

Humans stay on the critical decision path.

Approval is explicit. Execution is not allowed to blur past that step.

risks

Known failure modes

This is a workflow document, so the caveats belong on the page.

Context is not shared automatically

Pass files such as plan.md. Do not assume agent memory survives across boundaries.

Review loops can get expensive

Set exit conditions and turn limits so critique does not become churn.

Code leaves the machine

Both tools may send code to external APIs. Treat that as a real boundary.

Worker scope must stay narrow

Bound agents to specific file areas and avoid over-broad execution prompts.
